## 3. Método y resultado del delta del payload (🔒 chequeo central)

**Método.** Antes de editar, se extrajo el JSON del build vigente (Batch A+B1):
descompresión del blob `atob("…")` (base64 → gzip → JSON) → `/tmp/old_compact.json` (59 MB,
0 `nivel_alto`). Tras regenerar, se descomprimió el nuevo y se compararon **estructuralmente en
R** (`jsonlite::fromJSON` + `identical`), no por diff textual del JSON de 142 MB:
1. Igualdad de claves top-level.
2. `identical(payload_viejo, payload_nuevo)` **excluyendo** la clave `indicadores`.
3. Por cada indicador: igualdad de los campos compartidos + reporte de campos añadidos/quitados.

**Resultado.**
```
== claves top-level iguales: TRUE
== resto del payload (sin indicadores) IDENTICO: TRUE
== n indicadores old/new: 4 / 4
  ind 1 (Autoestima):    compartidos identicos=TRUE | añadidos={nivel_alto} | quitados={}
  ind 2 (Convivencia):   compartidos identicos=TRUE | añadidos={nivel_alto} | quitados={}
  ind 3 (Participación): compartidos identicos=TRUE | añadidos={nivel_alto} | quitados={}
  ind 4 (Hábitos):       compartidos identicos=TRUE | añadidos={nivel_alto} | quitados={}
```
Es decir: **todo el payload salvo `indicadores` es idéntico** (establecimientos, roster, ind,
dim, niv — todas las cifras `prom`/`dif`/`sigdif`/`difgru`/`sigdifgru`/`prom_gse` sin cambio); y
en `indicadores` el **único delta** es la aparición de `nivel_alto` en los 4, sin alterar
`definicion` ni ningún otro campo. **Ninguna cifra cambió.**
